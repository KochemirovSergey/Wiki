13. 3.6. Описание бизнес-модели проекта. Производственный план и план продаж

13.1. Описание бизнес-модели проекта (модель Остервальдера)

Блок

Описание

1. Потребительские сегменты (Customer Segments)

Граждане РФ — бесплатный доступ через opensource-версию и квоту 100 запросов в SaaS, источник данных и обратной связи.
НИИ и исследовательские университеты — 16 подтвержденных организаций (11 НИИ + 5 вузов), включая КФУ, ТГУ, ВШЭ, РАО, ИРПО, ФИПИ, ФИОКО.
Частные вузы и EdTech компании — 70 подтвержденных организаций (~50 частных вузов + ~20 EdTech, включая Skillbox, GeekBrains, Нетология, Синергия).
Федеральные органы управления — 8 подтвержденных ведомств (МинПрос, Минобрнауки, Рособрнадзор, Минтруд, Росмолодежь, РАО, ФОИВы, Правительство РФ).
Региональные органы управления — все 89 субъектов РФ, департаменты/министерства образования регионов.

2. Ценностные предложения (Value Proposition)

Граждане: полностью бесплатный доступ через opensource-версию + квота 100 запросов в SaaS, геймификация через опросы для получения дополнительных квот.
НИИ и вузы: готовая графовая БД с возможностью подключения собственного LLM API через Docker, цена 360 000 руб. (разово, 50% от стоимости создания БД).
EdTech и частные вузы: конкурентный анализ образовательного рынка, анализ трудоустройства выпускников, прогнозирование популярных программ, та же демократичная цена 36 000 руб./год.
Органы управления: комплексная аналитика для реализации Стратегии развития образования до 2040 года, локальное развертывание с обработкой внутренних документов, прогнозирование и мониторинг КПИ, интеграция с госсистемами, от 1 млн руб./год с ростом на 50% через дополнительные модули.

3. Каналы сбыта (Channels)

Академические каналы: публикации в научных журналах для создания академического веса, участие в конференциях (ММСО.EXPO, Архимед, Corporate Education Force).
Цифровые каналы: таргетированная реклама в образовательных Telegram-каналах, публикации на Habr.com, VC.ru.
Прямые продажи: использование существующих контактов ЛПР в органах управления (МФТИ, МЭИ, КФУ, ТГУ, Минпросвещения).
Продуктовые каналы: Telegram-бот, веб-портал с дашбордами, opensource-версия для самостоятельного развертывания.

4. Взаимоотношения с клиентами (Customer Relationships)

Граждане: самообслуживание через Telegram-бот, геймификация (получение квот за прохождение опросов), community-driven развитие продукта.
НИИ и образовательные организации: техническая поддержка, обучение работе с системой, регулярные консультации по использованию аналитики.
Крупные клиенты (органы управления): персональное сопровождение, кастомизация под специфику ведомства, регулярное предложение новых модулей для увеличения стоимости контракта на 50% ежегодно, консультации по стратегическим вопросам.

5. Потоки доходов (Revenue Streams)

Тариф 1 — Opensource: бесплатно, самостоятельная установка и интеграция собственного LLM/API — косвенная ценность через данные и отзывы пользователей.
Тариф 2 — Готовая БД для НИИ: 360 000 руб. (разово) за готовую графовую БД с лицензией и возможностью подключения собственного LLM API через Docker (50% от стоимости создания БД 720 000 руб.).
Тариф 3 — SaaS-решение: пополнение баланса по принципу 150% от затрат на API LLM + квота 100 бесплатных запросов для граждан, выбор модели LLM.
Тариф 4 — Локальное развертывание: от 1 млн руб./год для органов управления с планируемым ростом на 50% ежегодно через расширяемые контракты и дополнительные модули.

6. Ключевые ресурсы (Key Resources)

Технологические: собственная графовая база данных 750 таблиц статистических данных свзязанных между собой в графовой БД и обернуто в анлитику с агентами LLM.
Человеческие: команда разработчиков и аналитиков, экспертиза в области образовательной политики.
Инфраструктурные: облачная инфраструктура для SaaS, Docker-контейнеры для локального развертывания.
Партнерские: сеть прямых контактов с ЛПР в органах управления образованием, академическая репутация.

7. Ключевые виды деятельности (Key Activities)

Продуктовые: разработка и поддержка Telegram-бота, веб-портала, opensource-версии, интеграция с LLM API.
Аналитические: сбор и анализ образовательных данных, построение прогнозных моделей, создание отчетности для органов управления.
Маркетинговые: участие в конференциях, публикации в научных журналах, работа с академическим сообществом.
Продажные: прямые продажи органам управления, техническая поддержка клиентов, кастомизация решений.

8. Ключевые партнеры (Key Partners)

Технологические: провайдеры LLM API (Claude Sonnet 4.0 и др.), облачные провайдеры для инфраструктуры.
Академические: методическое и научное сообщество для создания академического веса, эксперты в области образовательной аналитики.
Медийные: профильные СМИ (РГ, РБК), образовательные Telegram-каналы, технологические платформы (Habr, VC.ru).
Государственные: органы управления образованием для внедрения в рамках Стратегии развития образования до 2040 года.

9. Структура издержек (Cost Structure)

Технологические: API LLM ($0.108/запрос ≈ 10.8 руб.), серверная инфраструктура на Selectel (123,600 руб./год), NLP preprocessing для оптимизации токенов, GPU-серверы для локальных развертываний (395,000-807,000 руб./месяц за H100).
Операционные: создание графовой БД (720,000 руб. единоразово), техническая поддержка клиентов, кастомизация решений для органов управления, обслуживание opensource-сообщества.
Административные: команда разработчиков и аналитиков, административные расходы, соответствие требованиям 152-ФЗ и госстандартам.
